Update From Older Version :
You can update a user's Blog to the newest version without loosing any of your data.

  1. Copy all of the new standalone "BlogX" release files into the "DATA" folder and copy the Database to the "DATASOURCE" folder.

  2. Force The "Config.asp" in the "DATA" folder to set the database parameter from the "DataFile.asp" which is created at create time
    (See example in the Conversion kit, or just copy the Config file across if it hasn't been updated in the new version)
    Then set the ExtraHTM.txt file to call "Count.asp" (again or just copy across the one in the Conversion kit)

  3. Login as an Admin go to "Create/Update" and insert the Blog name you wish to update.
    (The system will now automatically update all the files with the ones specified in the DATA folder)

  4. Open up the Old Database file and if need be, make the changes to update to the new version of the database.
